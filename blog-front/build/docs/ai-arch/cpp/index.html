c++排查问题工具栈​

调试​

一般使用gdb或者gdbserver。

远程服务器编译时，记得加-g参数 g++ helloworld.cpp -g 或者cmake增加 cmake -DCMAKE_BUILD_TYPE=Debug标记。

用gdbserver启动：（gdb打断点命令查文档） gdbserver :12345 ./a.out

问题排查工具​

工具作用
Valgrind / heaptrack查看内存泄漏问题、越界访问
perf方法追踪
perf + FlameGraph可以查火焰图
GDB附加到运行进程，查线程栈也可
strace跟踪系统调用
